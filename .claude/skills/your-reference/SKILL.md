# Your reference

Before writing, look at what the user already has. A new article should build on their
existing work, not contradict or duplicate it. This is what keeps a content library
coherent and well interlinked.

## Input

The keyword, plus the research brief from `content/1-research/{slug}.md`.

## How to run

Find the user's existing content on this and adjacent topics:

- Use the user's site (from `your-setup/brand-and-voice.md`). If Search Console is on,
  use it to find their ranking pages for related queries. Otherwise search their domain
  with built-in web search (for example a `site:` query).

## Steps

1. Identify the user's articles that touch this topic.
2. For each, pull out: useful framings or definitions worth staying consistent with,
   sections that could be linked to from the new piece, and any unique points or examples
   the user has already made that should not be re-explained from scratch.
3. Note good **internal linking** targets for the draft stage.
4. If `your-setup/your-offer.md` shows the user's content usually ties back to what they
   sell, flag the strongest in-context product angle their existing work uses.
5. **Proprietary-input pass.** This is what stops the article being "the average of the
   internet" that anyone could generate from a one-line prompt. Surface anything the user
   has that a competitor does not: original data or benchmarks, a contrarian POV, named
   customer examples, a framework or process they have published, screenshots of their own
   product or results. Pull these from their existing content and from `your-setup/`. Flag
   the strongest one or two for the draft to build a differentiated angle around. If the
   user has none on this topic, say so and suggest what original input would set the piece
   apart.

## Output

Write to `content/2-reference/{slug}.md`: a short list of relevant existing articles, the
elements worth reusing, the internal links to include, and the proprietary inputs that can
differentiate this piece. If the user has nothing on this topic yet, say so plainly and note
the article will set the reference for future ones.
